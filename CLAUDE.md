# 아재요 — 프로젝트 가이드

## 작업 규칙

- **스킬(impeccable 등)을 항상 사용할 것** — UI 생성/수정 시 관련 스킬이 있으면 반드시 활용
- 코드 작성 후 `npx next build`로 빌드 확인 필수
- **커밋 전 CLAUDE.md 업데이트** — 구조/기능 변경이 있으면 커밋 전에 CLAUDE.md를 먼저 수정할 것

---

## 프로젝트 개요

**아재요**는 기혼 남성(유부남)들을 위한 모바일 웹 커뮤니티 앱이다.
아내/장모님/가족에게 보낼 메시지 족보, 일정 관리, 선물 추천, 커뮤니티 기능을 제공한다.

- **기술 스택**: Next.js 14 (App Router), React 18, TypeScript, Tailwind CSS 3, shadcn/ui
- **폰트**: Noto Sans KR (Google Fonts)
- **모바일 퍼스트**: max-width 480px, 확대 방지, safe area 대응
- **현재 상태**: 프론트엔드 목업 (백엔드 없음, mock 데이터 사용)

## 브랜드 & 디자인

### 브랜드 성격
- **따뜻한, 유머러스, 편안한**
- 아재감성 — 진지하지 않고, 서로 공감하며 웃을 수 있는 분위기
- 감정 목표: 편안함, 소속감, 실용적 도움을 받는 만족감

### 디자인 원칙
1. **실용 우선** — 꾸미기보다 쓰기 편하게. 복사 한 번이면 끝나는 경험
2. **따뜻한 미니멀** — 요소는 최소화하되, 색감과 여백으로 온기를 전달
3. **부담 없는 인터랙션** — 터치 피드백은 자연스럽고 가볍게, 과한 애니메이션 지양
4. **모바일 퍼스트** — 터치 타겟 충분히 확보, 한 손 조작 최적화

### 색상 팔레트 (라이트 모드 전용)
CSS 변수는 HSL 값만 저장 (`22 60% 42%`), Tailwind 시맨틱 클래스에서 `hsl()` 래핑 처리됨.

| 토큰 | 값 | 용도 |
|------|-----|------|
| `--primary` | `22 60% 42%` | 테라코타 브라운, CTA/강조 |
| `--secondary` | `35 25% 93%` | 연한 배경, 비활성 버튼 |
| `--background` | `30 20% 97%` | 페이지 배경 |
| `--card` | `35 20% 99%` | 카드 배경 |
| `--foreground` | `30 15% 12%` | 본문 텍스트 |
| `--muted-foreground` | `30 10% 45%` | 보조 텍스트 |
| `--sage` | `150 20% 55%` | 초록(완료 상태 등) |
| `--amber` | `40 80% 60%` | 투표 뱃지 등 |

**중요**: `var(--primary)` 직접 사용 시 색상이 적용되지 않음. 반드시 Tailwind 시맨틱 클래스(`bg-primary`, `text-foreground` 등) 사용하거나, inline style에서는 `hsl(22 60% 42%)` 직접 입력.

### 카드 스타일
- border 없이 따뜻한 배경색(`hsl(36 30% 93%)`)으로 영역 구분
- `rounded-2xl`, 그림자 없음
- 카드 안의 카드(중첩) 금지 — 시각적 노이즈

## 앱 구조

### 인증 시스템
- **AuthContext** (`auth-context.tsx`): 전역 로그인 상태 관리 (React Context)
- **디폴트 상태**: 로그인됨 (`isLoggedIn: true`)
- **비로그인 시 동작**:
  - 홈: 다가오는 일정/추천 대신 가입 유도 배너 표시
  - 일정/마이페이지 탭: 토스트 메시지("로그인이 필요한 기능이에요") 후 `/login`으로 이동
  - 커뮤니티/족보: 비로그인에서도 접근 가능
- **ToastProvider** (`toast.tsx`): 전역 토스트 알림 시스템

### 네비게이션
- **하단 탭**: 홈, 커뮤니티, 일정, 족보, MY (5개)
  - 비로그인 시 일정/MY 탭 클릭 → 토스트 + 로그인 이동
- **헤더**: 페이지별 타이틀 + 좌측 글쓰기(연필) + 우측 알림(종)
  - 스크롤 시 sticky 헤더 (올릴 때 나타남, 내릴 때 숨김)
  - 글쓰기/알림 버튼은 `CustomEvent`로 이벤트 발행
- **AppShell** (`app-shell.tsx`): 경로에 따라 헤더/하단탭 표시 여부 결정
  - `/login`, `/signup` 경로: 헤더/하단탭 숨김
  - 그 외: 헤더/하단탭 표시

### 이벤트 시스템
| 이벤트 | 발행 위치 | 수신 위치 | 동작 |
|--------|-----------|-----------|------|
| `header:create` | `page-header.tsx` | 각 페이지 | 글쓰기/등록 바텀시트 오픈 |
| `header:notify` | `page-header.tsx` | `notification-listener.tsx` (레이아웃) | 알림 바텀시트 오픈 |

## 페이지별 기획

### 1. 홈 (`/`)
- **비로그인**: 가입 유도 배너 (아재 감성 카피 + 기능 태그 3개 + CTA)
- **로그인 시**:
  - **다가오는 일정**: 가로 스크롤 카드, D-7 이내는 primary 배경 강조
  - **추천**: 가장 가까운 일정의 태그와 연결된 선물/행동 추천 (최대 3개)
- **공통** (로그인 여부 무관):
  - **이달의 활동왕**: 이달 활동점수 TOP 3 유저 (가로 스크롤, 금/은/동 뱃지), 탭 시 유저 프로필 바텀시트
  - **인기 게시글**: 좋아요 순 상위 4개, 탭 시 게시글 상세 바텀시트 (내용, 투표 결과, 좋아요, 댓글)

### 2. 커뮤니티 (`/community`)
- **카테고리 탭**: 전체, 선물, 부부싸움, 어른들 취미, 육아, 생활꿀팁, 자유게시판
  - 스크롤 시 sticky 헤더에 축소 탭 표시
  - 카테고리 변경 시 스크롤 최상단 이동
- **게시글 카드**: 프로필, 제목, 내용(80자 초과 시 더보기), 이미지 캐러셀, 투표, 좋아요/댓글
- **투표**: A/B 투표, 퍼센티지 바 애니메이션, 재투표(토글) 가능
- **댓글**: 바텀시트, 시간순 정렬, 좋아요 가능
- **글쓰기**: 바텀시트
  - 일반 글: 카테고리 → 제목 → 내용 → 사진(최대 5장, 캐러셀 미리보기, 4:5/1:1 비율 선택)
  - 투표 글: 카테고리 → 질문 → A/B 선택지(VS 구분) → 부연 설명(선택)
  - 등록 시 피드 최상단에 추가

### 3. 일정 (`/schedule`) — 로그인 필요
- **예정된 일정 / 지난 일정** 섹션 분리
- **일정 카드**: 월/일 뱃지 + D-day + 태그 + 메모, 클릭 시 추천 바텀시트
- **추천 바텀시트**: 일정 태그 기반 선물/행동 추천 목록
- **일정 등록**: 바텀시트, 이름 + 날짜 + 태그(검색/생성 가능)

### 4. 족보 (`/jokbo`)
- **카테고리 탭**: 아내 생일 편지, 장모님 생신 카톡, 사과 문자, 기념일 메시지, 응원 한마디
- **템플릿 카드**: 편지지 스타일 (따뜻한 크림색 배경)
  - 콘텐츠가 주인공 — 제목/작성자는 하단 footer
  - 더보기/접기: max-height 토글 + 그래디언트 페이드 + 화살표 아이콘
  - 좋아요: 고스트 버튼, 활성화 시 연한 primary 배경 + primary 텍스트
  - 복사: primary 배경 버튼, 클릭 시 클립보드 복사 + sage 색상으로 "완료" 표시 (2초)

### 5. MY (`/mypage`) — 로그인 필요
- **프로필 카드**: 아바타, 이름, 결혼 정보, 등급 뱃지 (클릭 시 등급 안내 바텀시트), 활동 뱃지
- **등급 프로그레스 바**: 다음 등급까지 진행률 표시
- **활동 통계**: 게시글/좋아요/족보 기여 (3열 그리드)
- **메뉴 목록**:
  - 📝 내 게시글 → `/mypage/posts` (투표 결과 퍼센티지 바 포함)
  - 📋 내가 올린 족보 → `/mypage/jokbo` (펼침/접기, 복사 기능)
  - 🏅 등급 안내 → 바텀시트 (전체 등급 + 점수 획득 방법)
  - 🔔 알림 설정 → `/mypage/notification` (6개 항목 토글 스위치)
  - 👤 프로필 수정 → `/mypage/profile` (닉네임, 한줄소개)
  - ⚙️ 설정 → `/mypage/settings` (이용약관, 개인정보, 문의, 로그아웃, 회원탈퇴)

### 6. 로그인 (`/login`)
- 아재요 로고 + 브랜딩
- 소셜 로그인 3종: 카카오(노란색), 네이버(초록색), Google(흰색)
- 헤더/하단탭 없는 풀스크린 레이아웃

### 7. 회원가입 (`/signup`)
- **3단계 플로우** (상단 프로그레스 바):
  1. 약관동의: 전체동의/개별 체크 (필수 3개 + 선택 1개)
  2. 기본정보: 닉네임(2~12자), 결혼연도, 자녀 수
  3. 완료: 환영 메시지 + "시작하기" → 홈 이동 (login() 호출)

### 8. 알림 (바텀시트, 전역)
- 타입: 좋아요(❤️), 댓글(💬), 일정(📅), 시스템(🎉)
- 읽지 않은 알림: 따뜻한 배경 + primary 점 표시
- "모두 읽음" 일괄 처리

## 공통 UI 패턴

### 바텀시트 (`bottom-sheet.tsx`)
- 공통 `BottomSheet` 컴포넌트로 모든 바텀시트 통일
- 닫기 방법: **우측 상단 X 버튼** + 배경(오버레이) 탭
- `fixed inset-0 z-[60]`, `bg-black/40 backdrop-blur-sm` 오버레이
- `max-w-[480px] rounded-t-3xl`, 헤더에 드래그 핸들 + X 버튼 자동 포함
- `animate-fade-up` 진입 애니메이션
- props: `onClose`, `children`, `className?`, `style?`

### 카테고리 탭
- 가로 스크롤 pill 버튼, `scrollbar-hide`
- 활성: `bg-primary text-primary-foreground`, 비활성: `bg-secondary text-muted-foreground`
- 페이지 내 기본 크기 + sticky 헤더 축소 크기(`small` prop)

### 이미지 캐러셀 (`ImageCarousel`)
- 터치 기반 수동 스와이프 (방향 잠금: 가로 스와이프 시 페이지 세로 스크롤 방지)
- 하단 인디케이터 도트
- 4:5 또는 1:1 비율 지원

### 토스트 (`toast.tsx`)
- `fixed top-16` 상단 중앙, z-100
- 2초 표시 후 fade-out
- `useToast().show("메시지")` 로 어디서든 호출

### 애니메이션
- `animate-fade-up`: opacity 0→1 + translateY 12px→0, 0.5s ease-out
- `animationDelay`로 stagger 효과
- `active:scale-[0.97]` 또는 `active:scale-95` 터치 피드백
- `transition-all duration-200` 상태 전환

## 데이터 구조 (`src/data/mock.ts`)

### 주요 타입
- `UserGrade`: 등급 (level, name, emoji, minPoints)
- `User`: 유저 (id, name, initials, subtitle, gradeLevel, activityPoints, monthlyPoints, stats, badges)
- `Post`: 커뮤니티 게시글 (TEXT/VOTE), 이미지, 댓글, 투표 옵션 포함
- `Schedule`: 일정 (날짜, 태그, 메모)
- `ScheduleTag`: 시스템/사용자 태그 (색상 포함)
- `Template`: 족보 문구 (카테고리, 좋아요/복사 카운트)
- `Recommendation`: 태그 기반 추천 (순위, 이모지, 설명)
- `Category`: 선물 | 부부싸움 | 어른들 취미 | 육아 | 생활꿀팁 | 자유게시판
- `TemplateCategory`: 아내 생일 편지 | 장모님 생신 카톡 | 사과 문자 | 기념일 메시지 | 응원 한마디

### 주요 헬퍼 함수
- `getGrade(points)`: 점수에 해당하는 등급 반환
- `getTopMonthlyUsers(count)`: 이달 활동왕 TOP N 반환
- `getDday(dateStr)`: D-day 계산
- `getUpcomingSchedules()`: 다가오는 일정 목록
- `getRecommendationsForSchedule(schedule)`: 일정 기반 추천

### Mock 유저
- `currentUser`: 김아재 (로그인된 유저, 등급 4 베테랑 남편)
- `users`: 6명 (김아재, 살림왕남편, 딸바보파파, 워킹대디, 결혼5년차아빠, 로맨틱가이)
